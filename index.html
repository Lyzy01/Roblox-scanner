ENGINE DOCUMENTATION

How does the Safety Score work? +
Nexus analyzes variables such as creator account age relative to visit volume, metadata configuration properties, and public verification records to determine asset safety.
What constitutes a High Risk result? +
High risk ratings are commonly assigned to experiences built by brand new accounts exhibiting anomalous traffic (potential botting) or unvetted structures.
Can it evaluate private experiences? +
No. The security parser exclusively evaluates configuration structures exposed publicly through standard application programming interfaces.
How does Player Scan evaluate accounts? +
It cross-references the account's total active creation timeline against behavioral social markers like total friends, followers, and global verification configurations.
What signals a "Highly Suspicious" status? +
Accounts created within a critical 7-day threshold possessing entirely empty friend metrics or default configurations heavily mimic systemic bot distributions.
Does this look up hidden user details? +
No. Nexus strictly reads public architecture points exposed through the standardized open infrastructure layer of the official Roblox Web APIs.
